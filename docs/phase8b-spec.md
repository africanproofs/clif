## What clif is

clif is the African Proofs consumer that migrates the FTSO reward claimer
(`ftso-fee-claimer` / FlareOracle) off a local `.env PRIVATE_KEY=` and onto the
**fwd** signing daemon. This is **Phase 8b** of the fwd program: the first
deleted `.env PRIVATE_KEY=` line — the single most important unit of value the
program ships, and the event that lifts fwd's doctrine-ship freeze.

## The decision (authoritative)

Build clif as a **fully keyless** Python claimer:

- All discovery — claimable reward epochs, Merkle proofs from fsp-rewards — is
  keyless RPC + HTTP. clif holds **zero** private keys. There is no
  `.env PRIVATE_KEY=` anywhere in clif. If you ever need a local signing key,
  **STOP** — you have wandered out of scope.
- The **only** key operation is one transaction: `RewardManager.claim(...)`.
  clif builds the calldata and calls fwd `POST /v1/sign-and-send`. fwd holds
  the key, gates by policy, signs, broadcasts. clif never sees a key.
- `flare-foundation/signing-tool` / `SIGNING_POLICY_PRIVATE_KEY` is **out of
  scope** — deferred to **fwd Phase 9** (a structured protocol-message signer;
  structured/decodable intent, NOT raw `eth_sign`, NOT a local key). Do not
  port it, do not scaffold a disabled path, do not put its key in a `.env`.
  One README paragraph, nothing more.

## Non-negotiable constraints

1. **clif holds zero private keys.** Any `.env PRIVATE_KEY=` in clif is a
   regression. Enforced by `clif.config.assert_keyless()` + a forbidden-deps
   rule (`eth-account`, `eth-keys`, `pycryptodome`, `web3`, `argon2`).
2. **Never modify the fwd repo, never author fwd's `policy.yaml`.** If
   integration reveals fwd needs any change (missing ABI, endpoint gap, the
   `policy.example.yaml` defect), **STOP and report to the operator.** Do not
   "helpfully" edit fwd.
3. **Derive every on-wire value from the real producing path — never a doc or
   a mental model.** The `claim` calldata and the policy method-signature MUST
   come from running the actual claim-construction code against real
   fsp-rewards proofs. A hand-authored transaction shape is the defect class
   that has cost this program live-drill failures. Binding.
4. **fwd's `docs/policy.example.yaml` is VERIFIED WRONG for `claim`.** The
   correct canonical signature is in `docs/fwd-contract.md` §ABI. Do not copy
   the example; report it to the operator (fwd-side, not clif's to fix).
5. **Commits:** a single terse conventional line (`feat: update`, …) — no
   body, no specifics. **Never** add a `Co-Authored-By: Claude` / any AI
   co-author / "Generated with" line to any commit, PR, tag, or release —
   strip it if a tool adds one. Operator is sole author. Do not push if a
   remote block exists; ask the operator.
6. **Operator-gated (Core invariant #15, inherited).** You build and rehearse;
   the production Flare claim and the `.env` deletion happen only under
   explicit operator approval. Surface every deviation; never hide one; if the
   spec contradicts reality, STOP and report.

## Deliverables

1. **clif**: keyless Python claimer. Keyless epoch/proof discovery; the one
   key op = build real `RewardManager.claim` calldata → `POST
   /v1/sign-and-send` → poll `GET /v1/transactions/{tx_id}` to mined.
   Idempotency-Key per logical claim. Clean terminal-vs-retry handling.
2. **fwd integration spec artifact** (`docs/fwd-integration-spec.md`,
   generated by `clif spec`): exact chain id + RewardManager `to` per network;
   the real decoded intent (verified signature + scalar args, esp.
   `_recipient`); real captured calldata. The operator writes fwd's policy
   from this. This is the handshake.
3. **Delete the active `.env PRIVATE_KEY=`** from the fee-claimer path — only
   after the fwd-routed path is proven on the rehearsal ladder. This deletion
   is the Phase-8b deliverable.
4. **Rehearsal ladder**: prove the full path with clif-GENERATED calldata on
   **Coston2** → **Songbird** → **Flare** (production, operator-gated). See
   `docs/verification.md`.
5. **README**: one paragraph deferring signing-tool to fwd Phase 9.
